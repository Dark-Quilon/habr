# Обзор плана: Django блог + Docker + Redis

## Цель

Написать Django блог с нуля и запустить его через Docker с Redis.
В итоге весь проект стартует одной командой: `docker compose up --build`

## Что будет в итоге

- Django блог с авторизацией, статьями, комментариями, голосами
- Профили, подписки, лента, уведомления
- Поиск, сортировка, пагинация
- Redis кэш для статей
- Всё в Docker

## Структура файлов которые нужно создать

```
Habr/
├── Dockerfile              # образ Django приложения
├── docker-compose.yml      # описание всех сервисов
├── .env                    # переменные окружения (секреты)
├── .env.example            # пример .env без секретов
├── .dockerignore           # что не копировать в образ
├── requirements.txt        # все зависимости Python
├── mysite/
│   └── settings.py         # читает конфиг из .env
├── blog/
│   ├── models.py
│   ├── views.py
│   ├── urls.py
│   ├── forms.py
│   ├── cache.py
│   ├── context_processors.py
│   ├── templatetags/
│   │   └── markdown_extras.py
│   └── templates/
└── manage.py
```

## Порядок выполнения (с нуля до Docker)

2. `10` — модели, миграции, SQLite
3. `11` — авторизация

5. `13` — комментарии, голоса, профили, подписки, уведомления, Markdown

## Главный принцип

После каждого этапа сайт запускается и что-то делает.
Не пытаться сделать всё сразу.

# Этап 6: Поиск, сортировка, пагинация

## 1. Пагинация

Разбить список статей на страницы по 10 штук.

В `article_list` view:

```python
from django.core.paginator import Paginator

def article_list(request):
    articles = Article.objects.filter(status='published').select_related('author')
    paginator = Paginator(articles, 10)
    page_number = request.GET.get('page')
    page_obj = paginator.get_page(page_number)
    return render(request, 'blog/article_list.html', {'page_obj': page_obj})
```

В шаблоне:

```html
{% for article in page_obj %}
  ...
{% endfor %}

<!-- Навигация -->
{% if page_obj.has_previous %}
  <a href="?page={{ page_obj.previous_page_number }}">← Назад</a>
{% endif %}

Страница {{ page_obj.number }} из {{ page_obj.paginator.num_pages }}

{% if page_obj.has_next %}
  <a href="?page={{ page_obj.next_page_number }}">Вперёд →</a>
{% endif %}
```

## 2. Поиск

Поиск по заголовку, содержимому и тегам одновременно.

В `article_list` view добавить:

```python
from django.db.models import Q

def article_list(request):
    articles = Article.objects.filter(status='published').select_related('author')
    query = request.GET.get('q')

    if query:
        articles = articles.filter(
            Q(title__icontains=query) |
            Q(content__icontains=query) |
            Q(tags__name__icontains=query)
        ).distinct()

    paginator = Paginator(articles, 10)
    page_obj = paginator.get_page(request.GET.get('page'))
    return render(request, 'blog/article_list.html', {
        'page_obj': page_obj,
        'query': query,
    })
```

В шаблоне (в navbar или на странице):

```html
<form method="get" action="{% url 'article_list' %}">
  <input type="text" name="q" value="{{ query|default:'' }}" placeholder="Поиск...">
  <button type="submit">Найти</button>
</form>
```

## 4. Сортировка

Три варианта: новые, по рейтингу, популярные.

```python
from django.db.models import Sum, Case, When, IntegerField

def article_list(request):
    articles = Article.objects.filter(status='published').select_related('author')
    sort = request.GET.get('sort', 'new')

    if sort == 'rating':
        articles = articles.annotate(
            rating_score=Sum(Case(
                When(votes__value=1, then=1),
                When(votes__value=-1, then=-1),
                default=0,
                output_field=IntegerField()
            ))
        ).order_by('-rating_score', '-created_at')
    elif sort == 'popular':
        articles = articles.order_by('-views', '-created_at')
    else:
        articles = articles.order_by('-created_at')

    ...
    return render(request, 'blog/article_list.html', {
        'page_obj': page_obj,
        'sort': sort,
    })
```

В шаблоне:

```html
<a href="?sort=new">Новые</a>
<a href="?sort=rating">По рейтингу</a>
<a href="?sort=popular">Популярные</a>
```

## 5. Сохранять параметры при пагинации

Проблема: при переходе на страницу 2 теряется фильтр по тегу и поисковый запрос.

Решение — передавать все параметры в ссылках пагинации:

```html
{% if page_obj.has_previous %}
  <a href="?page={{ page_obj.previous_page_number }}&sort={{ sort }}&tag={{ tag.slug|default:'' }}&q={{ query|default:'' }}">← Назад</a>
{% endif %}
```

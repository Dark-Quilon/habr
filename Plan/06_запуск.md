# Шаг 6: Пошаговый запуск с нуля

## Предварительные требования

- Установлен Docker Desktop: https://www.docker.com/products/docker-desktop/
- Docker Desktop запущен (иконка в трее)

Проверить что Docker работает:
```bash
docker --version
docker compose version
```

---

## Шаг 1: Обновить requirements.txt

Заменить содержимое `requirements.txt`:

```
Django==6.0.3
Pillow
python-slugify
markdown
django-redis
gunicorn
```

Точные версии можно узнать командой:
```bash
.venv/Scripts/pip freeze
```

## Шаг 3: Обновить settings.py

Заменить начало файла (см. `05_settings.md` для полного кода):

```python
import os
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent.parent

SECRET_KEY = os.environ.get('SECRET_KEY', 'fallback')
DEBUG = os.environ.get('DEBUG', 'True') == 'True'
ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', 'localhost').split(',')
REDIS_URL = os.environ.get('REDIS_URL', 'redis://127.0.0.1:6379/1')
```

Заменить CACHES:
```python
CACHES = {
    "default": {
        "BACKEND": "django_redis.cache.RedisCache",
        "LOCATION": REDIS_URL,
        "OPTIONS": {"CLIENT_CLASS": "django_redis.client.DefaultClient"}
    }
}
DJANGO_REDIS_IGNORE_EXCEPTIONS = True
```

Добавить STATIC_ROOT:
```python
STATIC_ROOT = BASE_DIR / 'staticfiles'
```

## Шаг 4: Создать Dockerfile

Создать файл `Dockerfile` в корне проекта:

```dockerfile
FROM python:3.12-slim
WORKDIR /app
ENV PYTHONUNBUFFERED=1
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
RUN python manage.py collectstatic --noinput
EXPOSE 8000
CMD ["gunicorn", "mysite.wsgi:application", "--bind", "0.0.0.0:8000", "--workers", "2"]
```

## Шаг 6: Создать docker-compose.yml

Создать файл `docker-compose.yml` в корне проекта:

```yaml
services:

  redis:
    image: redis:7-alpine
    restart: always
    ports:
      - "6379:6379"

  web:
    build: .
    restart: always
    ports:
      - "8000:8000"
    env_file:
      - .env
    volumes:
      - ./media:/app/media
      - ./db.sqlite3:/app/db.sqlite3
    depends_on:
      - redis
    command: >
      sh -c "python manage.py migrate &&
             gunicorn mysite.wsgi:application --bind 0.0.0.0:8000 --workers 2"
```

## Шаг 7: Создать пустой db.sqlite3 (важно!)

Перед первым запуском нужно создать пустой файл базы данных.
Если его нет — Docker создаст папку вместо файла и migrate упадёт.

```bash
# Windows
type nul > db.sqlite3

# Mac/Linux
touch db.sqlite3
```

## Шаг 8: Запустить

```bash
docker compose up --build
```

Первый запуск займёт 2-5 минут — Docker скачает образы и установит зависимости.

Сайт будет доступен на: http://localhost:8000

## Возможные ошибки

### `collectstatic` падает с ошибкой
Добавить в `settings.py`:
```python
STATIC_ROOT = BASE_DIR / 'staticfiles'
```

### `django_redis` не найден
Убедиться что `django-redis` есть в `requirements.txt` (не в `.venv`).

### Сайт не открывается на localhost:8000
Проверить логи:
```bash
docker compose logs web
```

### Redis не подключается
Убедиться что в `.env` адрес `redis://redis:6379/1` (не `127.0.0.1`).
Внутри Docker контейнеры общаются по имени сервиса, не по localhost.

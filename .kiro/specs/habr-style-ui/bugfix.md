# Bugfix Requirements Document

## Introduction

UI сайта не соответствует требованиям дизайна в стиле Habr. Текущий интерфейс использует стандартную тёмную Bootstrap-шапку с логотипом "BlogApp" и текстовыми ссылками, тогда как требуется тёмно-синяя шапка в стиле Habr с иконочной навигацией. Главная страница не содержит секцию "Моя лента", а карточки статей не имеют кнопки "Читать дальше".

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN пользователь открывает любую страницу сайта THEN шапка отображается с чёрным фоном (`bg-dark`) вместо тёмно-синего цвета в стиле Habr

1.2 WHEN пользователь смотрит на шапку THEN логотип отображается как "BlogApp" вместо "Habr"

1.3 WHEN пользователь смотрит на шапку THEN навигация справа отображается в виде текстовых ссылок ("Войти", "Регистрация", "Лента", "Уведомления") вместо иконок (лупа, карандаш, аккаунт)

1.4 WHEN пользователь смотрит на шапку THEN слева отсутствует бургер-иконка (три горизонтальные полоски) для показа популярных тегов/тем

1.5 WHEN пользователь открывает главную страницу THEN отображается только список статей без секции "Моя лента" над ним

1.6 WHEN пользователь не авторизован и открывает главную страницу THEN секция "Моя лента" полностью отсутствует

1.7 WHEN пользователь просматривает карточку статьи THEN отсутствует кнопка "Читать дальше" для перехода к полному тексту

### Expected Behavior (Correct)

2.1 WHEN пользователь открывает любую страницу сайта THEN шапка SHALL отображаться с тёмно-синим фоном в стиле Habr (например, `#1f2937` или аналогичный)

2.2 WHEN пользователь смотрит на шапку THEN логотип SHALL отображаться как "Habr"

2.3 WHEN пользователь смотрит на шапку THEN справа SHALL отображаться иконка лупы (поиск), иконка карандаша (написать статью) и иконка аккаунта (или кнопка "Войти" для неавторизованных)

2.3.1 WHEN пользователь нажимает на иконку лупы THEN SHALL открываться строка поиска (inline или выпадающая) для ввода поискового запроса

2.3.2 WHEN пользователь наводит курсор на иконку карандаша THEN SHALL отображаться tooltip с текстом "Написать публикацию"

2.4 WHEN пользователь смотрит на шапку THEN слева SHALL отображаться бургер-иконка (три горизонтальные полоски), при нажатии на которую показываются популярные теги/темы

2.5 WHEN авторизованный пользователь нажимает на иконку аккаунта THEN SHALL появляться выпадающее меню с опциями профиля и выхода

2.6 WHEN неавторизованный пользователь нажимает на иконку аккаунта THEN SHALL появляться выпадающее меню с опцией регистрации и входа

2.7 WHEN пользователь открывает главную страницу THEN SHALL отображаться секция "Моя лента" над списком статей

2.8 WHEN пользователь не авторизован и открывает главную страницу THEN секция "Моя лента" SHALL отображаться с сообщением о необходимости войти или быть пустой

2.9 WHEN пользователь просматривает карточку статьи THEN SHALL отображаться кнопка "Читать дальше", ведущая на страницу полного текста статьи

### Unchanged Behavior (Regression Prevention)

3.1 WHEN пользователь нажимает на логотип в шапке THEN система SHALL CONTINUE TO перенаправлять на главную страницу

3.2 WHEN авторизованный пользователь нажимает "Выйти" THEN система SHALL CONTINUE TO выполнять выход и перенаправлять на страницу входа

3.3 WHEN пользователь нажимает на иконку карандаша (написать статью) THEN система SHALL CONTINUE TO перенаправлять на страницу создания новой статьи

3.4 WHEN пользователь нажимает на иконку лупы THEN система SHALL CONTINUE TO предоставлять функциональность поиска статей

3.5 WHEN пользователь нажимает на тег в карточке статьи THEN система SHALL CONTINUE TO фильтровать список статей по выбранному тегу

3.6 WHEN пользователь нажимает на заголовок статьи в карточке THEN система SHALL CONTINUE TO переходить на страницу детального просмотра статьи

3.7 WHEN авторизованный пользователь имеет непрочитанные уведомления THEN система SHALL CONTINUE TO отображать счётчик непрочитанных уведомлений

3.8 WHEN пользователь использует пагинацию на главной странице THEN система SHALL CONTINUE TO корректно переключать страницы списка статей

# Habr Style UI Bugfix Design

## Overview

Текущий UI не соответствует дизайну в стиле Habr. Баг затрагивает три компонента:
- `Navbar.tsx` — неверный цвет фона, логотип "BlogApp", текстовые ссылки вместо иконок, отсутствует бургер-кнопка слева
- `app/page.tsx` — отсутствует секция "Моя лента" над списком статей
- `ArticleCard.tsx` — отсутствует кнопка "Читать дальше"

Стратегия исправления: минимальные точечные изменения в трёх файлах + добавление SCSS-переменной для тёмно-синего цвета шапки. Все существующие функции (поиск, теги, пагинация, уведомления, выход) сохраняются без изменений.

## Glossary

- **Bug_Condition (C)**: Условие, при котором UI не соответствует дизайну Habr — любой из трёх компонентов рендерится в дефектном виде
- **Property (P)**: Ожидаемое поведение при выполнении условия баги — компонент рендерится согласно требованиям Habr-стиля
- **Preservation**: Существующая функциональность (навигация, поиск, теги, пагинация, уведомления, выход), которая не должна измениться после фикса
- **Navbar**: Компонент `frontend/src/components/Navbar.tsx` — шапка сайта, присутствует на всех страницах
- **ArticleCard**: Компонент `frontend/src/components/ArticleCard.tsx` — карточка статьи в списке
- **HomePage**: `frontend/src/app/page.tsx` — главная страница со списком статей
- **bg-dark**: Bootstrap-класс, задающий чёрный фон (`#212529`) — текущий (дефектный) цвет шапки
- **habr-navbar**: Новый CSS-класс с тёмно-синим фоном `#1f2937`, заменяющий `bg-dark`

## Bug Details

### Bug Condition

Баг проявляется при рендере трёх компонентов. Каждый из них нарушает требования дизайна независимо.

**Formal Specification:**
```
FUNCTION isBugCondition(component)
  INPUT: component — один из ['Navbar', 'HomePage', 'ArticleCard']
  OUTPUT: boolean

  IF component = 'Navbar' THEN
    RETURN navbarHasClass('bg-dark')
           OR logoText = 'BlogApp'
           OR NOT hasIconSearch()
           OR NOT hasIconWrite()
           OR NOT hasIconAccount()
           OR NOT hasBurgerLeft()

  IF component = 'HomePage' THEN
    RETURN NOT hasMyFeedSection()

  IF component = 'ArticleCard' THEN
    RETURN NOT hasReadMoreButton()

  RETURN false
END FUNCTION
```

### Examples

- **Navbar фон**: Шапка рендерится с классом `bg-dark` (чёрный `#212529`) → ожидается тёмно-синий `#1f2937`
- **Логотип**: Текст логотипа "BlogApp" → ожидается "Habr"
- **Иконки**: Справа отображаются текстовые ссылки "Войти", "Регистрация" → ожидаются SVG-иконки лупы, карандаша, аккаунта
- **Бургер**: Слева нет кнопки с тремя полосками для тегов → ожидается бургер-кнопка
- **Моя лента**: На главной странице нет секции "Моя лента" → ожидается секция над списком статей (пустая или с сообщением для неавторизованных)
- **Читать дальше**: В карточке статьи нет кнопки → ожидается кнопка-ссылка на страницу статьи

## Expected Behavior

### Preservation Requirements

**Unchanged Behaviors:**
- Клик по логотипу в шапке продолжает перенаправлять на главную страницу (`/`)
- Авторизованный пользователь может выйти через меню аккаунта — выход и редирект на `/login` работают как прежде
- Иконка карандаша ведёт на `/articles/new` (создание статьи)
- Иконка лупы открывает строку поиска — поиск статей работает как прежде
- Клик по тегу в карточке статьи фильтрует список статей
- Клик по заголовку статьи переходит на страницу детального просмотра
- Счётчик непрочитанных уведомлений отображается для авторизованных пользователей
- Пагинация на главной странице корректно переключает страницы

**Scope:**
Все взаимодействия, не связанные с визуальным оформлением шапки, секцией "Моя лента" и кнопкой "Читать дальше", остаются полностью неизменными.

## Hypothesized Root Cause

1. **Неверный Bootstrap-класс в Navbar**: Используется `bg-dark` вместо кастомного класса с цветом `#1f2937`. Нет переменной SCSS для цвета шапки Habr.

2. **Устаревший текст логотипа**: Хардкод строки "BlogApp" в `Navbar.tsx` — не обновлялся при смене концепции дизайна.

3. **Текстовые ссылки вместо иконок**: Навигационные элементы реализованы как текстовые `<Link>` и `<button>`, SVG-иконки не добавлены. Бургер-кнопка слева отсутствует — не была предусмотрена в исходной реализации.

4. **Отсутствие секции "Моя лента" на главной**: `app/page.tsx` не содержит компонента или блока для отображения ленты подписок. Страница `/feed` существует отдельно, но не интегрирована в главную.

5. **Отсутствие кнопки в ArticleCard**: Компонент не содержит элемента для перехода к полному тексту — только заголовок-ссылка.

## Correctness Properties

Property 1: Bug Condition - Navbar соответствует дизайну Habr

_For any_ рендера компонента `Navbar`, исправленный компонент SHALL отображать тёмно-синий фон (`#1f2937`), логотип "Habr", бургер-кнопку слева, иконки лупы/карандаша/аккаунта справа вместо текстовых ссылок.

**Validates: Requirements 2.1, 2.2, 2.3, 2.3.1, 2.3.2, 2.4, 2.5, 2.6**

Property 2: Bug Condition - Главная страница содержит секцию "Моя лента"

_For any_ рендера главной страницы (`HomePage`), исправленный компонент SHALL отображать секцию "Моя лента" над списком статей. Для неавторизованных пользователей секция SHALL отображаться с сообщением о необходимости войти или быть пустой.

**Validates: Requirements 2.7, 2.8**

Property 3: Bug Condition - ArticleCard содержит кнопку "Читать дальше"

_For any_ рендера компонента `ArticleCard` с любой статьёй, исправленный компонент SHALL отображать кнопку "Читать дальше", ведущую на страницу `/articles/{slug}`.

**Validates: Requirements 2.9**

Property 4: Preservation - Существующая функциональность не нарушена

_For any_ взаимодействия пользователя, не связанного с исправляемыми дефектами (клик по логотипу, выход, поиск, теги, пагинация, уведомления, переход к статье), исправленный код SHALL производить точно такое же поведение, как оригинальный код.

**Validates: Requirements 3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 3.7, 3.8**

## Fix Implementation

### Changes Required

**File 1**: `frontend/src/styles/_variables.scss`

**Specific Changes**:
1. Добавить переменную Bootstrap `$navbar-dark-bg` или кастомный SCSS-класс `.habr-navbar` с `background-color: #1f2937`

---

**File 2**: `frontend/src/components/Navbar.tsx`

**Specific Changes**:
1. **Цвет фона**: Заменить класс `bg-dark` на `habr-navbar` (или inline-стиль `style={{ backgroundColor: '#1f2937' }}`)
2. **Логотип**: Заменить текст "BlogApp" на "Habr"
3. **Бургер слева**: Добавить кнопку с иконкой трёх полосок (Bootstrap `navbar-toggler` или кастомная кнопка) в левую часть navbar, при клике показывать offcanvas/dropdown с тегами
4. **Иконка лупы**: Заменить текстовую ссылку поиска на SVG-иконку лупы; при клике показывать inline-строку поиска (collapse или state toggle)
5. **Иконка карандаша**: Заменить ссылку на создание статьи на SVG-иконку карандаша с Bootstrap tooltip "Написать публикацию"
6. **Иконка аккаунта**: Для авторизованных — SVG-иконка аккаунта с dropdown (профиль, уведомления, выход); для неавторизованных — кнопка "Войти" или иконка с dropdown (войти, регистрация)

---

**File 3**: `frontend/src/app/page.tsx`

**Specific Changes**:
1. **Секция "Моя лента"**: Добавить блок над `<ArticleList>`. Использовать `getServerToken()` для определения авторизации. Если авторизован — показать ссылку на `/feed` или краткий превью ленты. Если нет — показать сообщение "Войдите, чтобы видеть свою ленту" со ссылкой на `/login`

---

**File 4**: `frontend/src/components/ArticleCard.tsx`

**Specific Changes**:
1. **Кнопка "Читать дальше"**: Добавить `<Link href={/articles/${article.slug}} className="btn btn-outline-primary btn-sm mt-2">Читать дальше</Link>` в нижнюю часть `card-body`

## Testing Strategy

### Validation Approach

Двухфазный подход: сначала зафиксировать дефектное поведение на нефиксированном коде (exploratory), затем верифицировать исправление и отсутствие регрессий.

### Exploratory Bug Condition Checking

**Goal**: Подтвердить наличие дефектов до внесения изменений. Зафиксировать counterexamples.

**Test Plan**: Написать тесты рендера компонентов на нефиксированном коде и убедиться, что они падают с ожидаемыми ошибками.

**Test Cases**:
1. **Navbar background test**: Рендер `<Navbar>` → проверить наличие класса `habr-navbar` или цвета `#1f2937` (упадёт на нефиксированном коде — есть только `bg-dark`)
2. **Navbar logo test**: Рендер `<Navbar>` → проверить текст "Habr" (упадёт — есть "BlogApp")
3. **Navbar icons test**: Рендер `<Navbar>` → проверить наличие SVG-иконок лупы и карандаша (упадёт — есть текстовые ссылки)
4. **Burger button test**: Рендер `<Navbar>` → проверить наличие бургер-кнопки слева (упадёт — отсутствует)
5. **MyFeed section test**: Рендер `<HomePage>` → проверить наличие секции "Моя лента" (упадёт — отсутствует)
6. **ReadMore button test**: Рендер `<ArticleCard article={mockArticle}>` → проверить наличие кнопки "Читать дальше" (упадёт — отсутствует)

**Expected Counterexamples**:
- `expect(getByText('Habr')).toBeInTheDocument()` — FAIL (найдено "BlogApp")
- `expect(container.querySelector('.habr-navbar')).toBeTruthy()` — FAIL (найден `bg-dark`)
- `expect(getByText('Читать дальше')).toBeInTheDocument()` — FAIL (элемент отсутствует)

### Fix Checking

**Goal**: Верифицировать, что для всех входных данных, где выполняется условие баги, исправленный код производит ожидаемое поведение.

**Pseudocode:**
```
FOR ALL component WHERE isBugCondition(component) DO
  result := render_fixed(component)
  ASSERT expectedBehavior(result)
END FOR
```

### Preservation Checking

**Goal**: Верифицировать, что для всех взаимодействий, не связанных с дефектами, исправленный код ведёт себя идентично оригинальному.

**Pseudocode:**
```
FOR ALL interaction WHERE NOT isBugCondition(interaction.component) DO
  ASSERT original_behavior(interaction) = fixed_behavior(interaction)
END FOR
```

**Testing Approach**: Property-based тестирование рекомендуется для preservation checking:
- Генерирует множество вариантов статей и проверяет, что теги, заголовки, рейтинги рендерятся корректно
- Покрывает граничные случаи (статья без тегов, длинный заголовок, нулевой рейтинг)
- Даёт сильные гарантии неизменности поведения для всех не-дефектных входных данных

**Test Cases**:
1. **Logo link preservation**: Клик по логотипу "Habr" → href="/" не изменился
2. **Write article link preservation**: Иконка карандаша → href="/articles/new" не изменился
3. **Search functionality preservation**: Поиск через иконку лупы → параметр `search` передаётся корректно
4. **Tag filter preservation**: Клик по тегу в `ArticleCard` → фильтрация работает как прежде
5. **Article title link preservation**: Клик по заголовку → переход на `/articles/{slug}` работает
6. **Notification badge preservation**: Счётчик уведомлений отображается для авторизованных пользователей
7. **Pagination preservation**: Пагинация на главной странице работает корректно

### Unit Tests

- Рендер `Navbar` с авторизованным пользователем: проверить наличие иконок, dropdown меню аккаунта
- Рендер `Navbar` без авторизации: проверить кнопку "Войти", отсутствие dropdown профиля
- Рендер `ArticleCard` с mock-статьёй: проверить наличие кнопки "Читать дальше" с корректным href
- Рендер `ArticleCard` без тегов: кнопка "Читать дальше" всё равно присутствует
- Рендер `HomePage` без авторизации: секция "Моя лента" отображается с сообщением о входе

### Property-Based Tests

- Генерировать случайные объекты `ArticleList` (разные slug, title, tags) → кнопка "Читать дальше" всегда присутствует и href всегда равен `/articles/{slug}`
- Генерировать случайные наборы тегов → теги в `ArticleCard` всегда рендерятся с корректными ссылками (preservation)
- Генерировать случайные значения `unreadCount` → badge уведомлений отображается корректно для авторизованных пользователей (preservation)

### Integration Tests

- Полный флоу: открыть главную страницу → увидеть секцию "Моя лента" → нажать "Читать дальше" на карточке → перейти на страницу статьи
- Флоу поиска: нажать иконку лупы в шапке → ввести запрос → список статей фильтруется
- Флоу авторизации: нажать иконку аккаунта → войти → шапка обновляется, показывает профиль и уведомления

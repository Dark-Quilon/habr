# Отчёт 1: Docker-инфраструктура

## Статус: ВЫПОЛНЕНО ✅

## Что сделано

### .gitignore
Создан файл `.gitignore` в корне проекта. Исключает из git:
- `.env` — секреты не попадают в репозиторий
- `db.sqlite3` — база данных
- `media/`, `staticfiles/` — загруженные файлы и статика
- `__pycache__/`, `*.pyc`, `*.pyo` — кэш Python
- `.venv/`, `venv/` — виртуальное окружение

### .env и .env.example
- `.env.example` — шаблон с переменными без реальных значений (безопасно коммитить)
- `.env` — реальные значения для локальной разработки (в git не попадает)

Переменные:
| Переменная | Значение (dev) | Назначение |
|-----------|---------------|-----------|
| SECRET_KEY | insecure-local-dev-key | Секретный ключ Django |
| DEBUG | True | Режим отладки |
| ALLOWED_HOSTS | localhost,127.0.0.1 | Разрешённые хосты |
| REDIS_URL | redis://redis:6379/1 | Адрес Redis внутри Docker |
| DATABASE_URL | sqlite:///db.sqlite3 | База данных |

### .dockerignore
Исключает из Docker-образа:
- `.venv`, `venv` — не копировать виртуальное окружение (сотни МБ)
- `__pycache__`, `*.pyc`, `*.pyo` — кэш Python
- `.git`, `.gitignore` — git-данные
- `db.sqlite3` — база данных хоста
- `media/` — медиафайлы хоста
- `Plan/` — документация плана
- `.env` — секреты

## Проблемы и решения

**Проблема:** Docker Desktop не запускался — не был установлен WSL2.
**Решение:** Установлен WSL2 с Ubuntu 24.04 через `wsl --install -d Ubuntu`, перезагрузка компьютера.

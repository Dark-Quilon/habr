# ЧАСТЬ 2: DOCKER ИНФРАСТРУКТУРА

## docker-compose.yml — 4 сервиса

### frontend
- Собирается из `frontend/Dockerfile` (node:20-alpine)
- Порт: 3000
- Зависит от: web
- Environment: NEXT_PUBLIC_API_URL=http://localhost:8000

### web (Django)
- Собирается из корневого `Dockerfile` (python:3.12-slim)
- Порт: 8000
- Зависит от: redis
- Автоматические миграции при старте
- Gunicorn с 2 воркерами
- Тома: media, staticfiles, db.sqlite3

### redis
- Образ: redis:7-alpine
- Порт: 6379
- Лимит памяти: 256 МБ
- Политика: allkeys-lru
- Без сохранения на диск

### gitlab
- Образ: gitlab/gitlab-ce:latest
- Порты: 8080 (HTTP), 8443 (HTTPS), 2222 (SSH), 5050 (Registry)
- Volume: gitlab_config, gitlab_logs, gitlab_data
- Healthcheck с интервалом 60с

### gitlab-runner
- Образ: gitlab/gitlab-runner:latest
- Зависит от: gitlab
- Volume: gitlab_runner_config

## Dockerfile (бэкенд)

```dockerfile
FROM python:3.12-slim
ENV PYTHONUNBUFFERED=1
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
RUN python manage.py collectstatic --noinput
EXPOSE 8000
CMD ["gunicorn", "mysite.wsgi:application", "--bind", "0.0.0.0:8000", "--workers", "2"]
```

Оптимизация: requirements.txt копируется первым — слой кэшируется при изменении только кода.

4. `router.push('/'); router.refresh()` → Navbar обновляется

# ЧАСТЬ 8: ПУБЛИЧНЫЙ ДОСТУП

## Туннель (localtunnel)

Проблема: ngrok заблокирован, cloudflared нестабилен.
Решение: `npx localtunnel --port 3000`

### Архитектурное решение

Next.js rewrite proxy в `next.config.js`:
```js
rewrites() {
  return [{
    source: '/api/v1/:path*',
    destination: 'http://127.0.0.1:8000/api/v1/:path*',
  }]
}
```

Клиентский API использует относительный URL (`/api/v1`), серверный — абсолютный (`http://127.0.0.1:8000/api/v1`).

Результат: один туннель на порт 3000 — и фронт и бэкенд работают.

# ЧАСТЬ 12: ИЗВЕСТНЫЕ ПРОБЛЕМЫ И НЕДОСТАТКИ

## Безопасность

| Проблема | Описание |
|---------|---------|
| XSS в Markdown | `markdownify` использует `mark_safe` без санитизации — нет bleach/nh3 |
| Rate limiting | Нет ограничения на количество запросов |
| SECURE_* настройки | Нет HSTS, SSL redirect для продакшена |

## Функциональность

| Проблема | Описание |
|---------|---------|
| Черновики недоступны по URL | ArticleDetailView фильтрует только published |
| Нет превью в списке | Нет краткого описания статьи |
| Нет счётчика комментариев в списке | Не отображается количество |
| Нет RSS/Atom ленты | Нет машиночитаемой подписки |
| Нет пагинации в my_articles | Все статьи без пагинации |
| Нет пагинации в profile | Все статьи автора без пагинации |

## Инфраструктура

| Проблема | Описание |
